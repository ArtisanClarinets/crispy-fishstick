# CHK-CARE-015: Documentation Review Checklist

**Document ID:** VS-CARE-CHK-015  
**Version:** 1.0.0  
**Effective Date:** 2026-02-21  
**Owner:** Delivery Lead  
**Status:** Active  
**Reading Level:** 8th Grade  
**Estimated Time:** 2-4 hours (monthly), 4-6 hours (quarterly deep clean)  
**Risk Rating:** Medium (stale documentation causes delays and errors)

**Goal:** Maintain current, accurate documentation that enables fast recovery, smooth transitions, and informed decisions through regular freshness verification.

---

## 1. Inventory of Documentation

- [ ] Document list current (all known docs catalogued)
- [ ] Ownership assigned (each doc has a named owner)
- [ ] Locations documented (where each file lives)
- [ ] Access permissions verified (right people can access)
- [ ] Version control confirmed (change history tracked)

**Time Estimate:** 20-30 minutes  
**Owner:** Delivery Lead

---

## 2. Freshness Assessment

For each document, determine status:

| Status | Definition | Action Required |
|--------|------------|-----------------|
| **Current** | Updated within 30 days | None |
| **Aging** | Updated 31-60 days ago | Review for needed updates |
| **Stale** | Updated 61-90 days ago | Update or schedule refresh |
| **Expired** | Updated 90+ days ago | Must update or archive |

- [ ] Each document assessed against above matrix
- [ ] Freshness scores recorded (TPL-CARE-012)
- [ ] Aging/stale/expired items flagged
- [ ] Update priorities ranked (by criticality)

**Time Estimate:** 30-45 minutes  
**CMMC Alignment:** CM.2.1 (baseline configurations), CM.3.1 (change control)

---

## 3. Critical Document Review

High-priority documents (update first if aging):

- [ ] **Inventory Snapshot** (TPL-CARE-010) - accurate asset list?
- [ ] **Access Register** (TPL-CARE-009) - current users/permissions?
- [ ] **Network Diagrams** - topology still correct?
- [ ] **Escalation Tree** - contacts still current?
- [ ] **Runbooks** - procedures still accurate?
- [ ] **Backup Procedures** - restore steps current?
- [ ] **Vendor Register** (TPL-CARE-011) - support info current?

**Time Estimate:** 45-60 minutes  
**Vantus standard:** requiring monthly review of all critical docs

---

## 4. Change Correlation

- [ ] Change log reviewed (what changed this month)
- [ ] Documents cross-referenced (which docs affected by changes)
- [ ] Updates triggered (docs marked for refresh)
- [ ] New docs identified (changes created need for new documentation)
- [ ] Obsolete docs flagged (systems removed, docs no longer needed)

**Time Estimate:** 20-30 minutes

---

## 5. Update Execution

- [ ] Critical aging documents updated
- [ ] Network diagrams refreshed (if topology changed)
- [ ] Asset inventory corrected (new/removed systems)
- [ ] Contact lists verified (email/phone still valid)
- [ ] Procedures tested (runbook steps still work)
- [ ] Screenshots updated (if UI changed)
- [ ] Links verified (no broken references)

**Time Estimate:** 60-120 minutes (varies by update volume)

---

## 6. Quarterly Deep Clean (Advanced+ Tiers)

Additional quarterly activities:

- [ ] Comprehensive review of all documentation
- [ ] Accuracy verification (does doc match reality?)
- [ ] Client accessibility check (can client find key docs?)
- [ ] Template standardization (consistent formatting)
- [ ] Naming convention audit (files follow standards)
- [ ] Archive of obsolete docs (old versions stored)
- [ ] Documentation gaps analysis (what's missing?)

**Time Estimate:** 2-3 hours (quarterly)  
**CMMC Alignment:** SC.1.1 (boundary protection), SC.2.1 (security architecture)

---

## 7. Quality Verification

- [ ] Technical accuracy confirmed (subject matter expert review)
- [ ] Completeness check (all required sections present)
- [ ] Clarity verified (8th grade reading level, no jargon)
- [ ] Format consistency (follows templates)
- [ ] Version numbers updated (incremented appropriately)
- [ ] Change log updated (what changed, when, why)

**Time Estimate:** 30-45 minutes

---

## 8. Reporting & Communication

- [ ] Freshness score published (TPL-CARE-012 completed)
- [ ] Monthly scorecard updated (doc status included)
- [ ] Client notified of significant updates ("Your network diagram is current")
- [ ] Stale items escalated (if updates not happening)
- [ ] Improvement opportunities logged (process enhancements)
- [ ] Next review scheduled (calendar entry created)

**Time Estimate:** 15-20 minutes  
**Owner:** Delivery Lead (execution), Account Manager (client communication)

---

## Documentation Freshness Scorecard

| Document Category | Target Freshness | Review Frequency |
|-------------------|------------------|------------------|
| Critical runbooks | Current (≤30 days) | Monthly |
| Asset inventory | Current (≤30 days) | Monthly |
| Network diagrams | Aging max (≤60 days) | Monthly |
| Vendor contacts | Current (≤30 days) | Monthly |
| Access registers | Current (≤30 days) | Monthly |
| General procedures | Aging max (≤60 days) | Quarterly |
| Training materials | Stale max (≤90 days) | Quarterly |

---

## Success Criteria

| Criterion | Target | Verification |
|-----------|--------|--------------|
| Critical docs current | 95%+ | Freshness score |
| Zero expired critical docs | 100% | Audit |
| Update completion | 90%+ on schedule | Tracking |
| Client accessibility | Verified quarterly | Test search |
| Accuracy complaints | <2% | Feedback |
