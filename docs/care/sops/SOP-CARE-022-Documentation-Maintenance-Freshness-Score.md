## What This Document Is

## Why Documentation Matters

Good documentation:
- Speeds up problem resolution
- Enables smooth handoffs
- Preserves institutional knowledge
- Supports compliance

## Step-by-Step: Documentation Management

### Step 1: Define Required Documents
By service tier, minimum includes:
| Document | Purpose |
|----------|---------|
| **Inventory** | What you have |
| **Access Register** | Who can access what |
| **Runbook Pointers** | How to handle common situations |
| **Escalation Tree** | Who to call for what |

### Step 2: Update with Changes
After any material change (SOP-CARE-017):
- Update relevant documents
- Record change date
- Notify stakeholders

### Step 3: Monthly Review
Review freshness score table:
| Status | Definition |
|--------|------------|
| **Current** | Updated within 30 days |
| **Aging** | Updated 30-60 days ago |
| **Stale** | Updated 60+ days ago |

Update stale files.

### Step 4: Quarterly Deep Clean (Advanced+)
- Comprehensive review
- Verify accuracy
- Confirm client can locate documents
- Update templates if needed

### Step 5: Report Status
Include in communications:
- Doc freshness summary
- Any gaps identified
- Planned updates

## Real Example

**Scenario**: New VPN added to network.

**Documentation Process**:
1. VPN configured and tested
2. Update network diagram
3. Update access documentation
4. Update runbook with VPN procedures
5. Update freshness score
6. Include in next scorecard: "What Changed" section
7. Notify client of documentation updates
