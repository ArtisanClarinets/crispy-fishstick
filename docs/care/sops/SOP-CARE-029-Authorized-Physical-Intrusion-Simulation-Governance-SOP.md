## What This Document Is

This SOP governs authorized physical security testing. We test facility security with explicit written permission, safety plans, and legal approvals controlled by you.

---

## Important: Rules Only

This document covers **governance and rules only**—not physical testing methods. We define:
- Authorization requirements
- Safety controls
- Reporting standards

**We do not provide tactical intrusion instructions.**

## Why Physical Testing Matters

Controlled physical testing:
- Identifies facility security gaps
- Tests access controls
- Validates security procedures
- Complements digital security

## Step-by-Step: Authorized Physical Testing

### Step 1: Written Approval Required
Must have:
- Signed written authorization
- ROE Pack specifying physical scope
- Do-not-do list clearly defined

### Step 2: Safety Plan Required
Document must include:
| Element | Description |
|---------|-------------|
| **Stop Authority** | Who can halt testing |
| **Emergency Contacts** | Who to call if issues |
| **Non-Harm Constraints** | No injury, no damage rules |

### Step 3: Validate Boundaries
Confirm:
| Check | Purpose |
|-------|---------|
| Do-not-touch areas defined | Protect sensitive areas |
| Prohibited actions listed | Prevent harm |
| Time windows agreed | Minimize disruption |
| Legal approvals confirmed | Ensure compliance |

### Step 4: Execute Only as Authorized
Testing boundaries:
- Follow approved scope exactly
- Stop immediately if any safety issue
- No deviation from plan

### Step 5: Stop if Safety at Risk
Immediate halt if:
- Safety constraint at risk
- Unexpected situation
- Stop authority invokes halt

### Step 6: Deliver Findings
Report focuses on:
| Area | Content |
|------|---------|
| **Policy Improvements** | Better procedures |
| **Process Gaps** | Missing controls |
| **Facility Controls** | Physical security enhancements |

---

## Hard Limits

| Never Allowed | Reason |
|---------------|--------|
| **Forced Entry** | Damage and legal risk |
| **Harm** | Safety first |
| **Coercion** | Ethical violation |
| **Without Legal Approval** | Client responsibility |

## Real Example

**Scenario**: Office building access control test.

**Testing Process**:
1. Written authorization obtained
2. ROE Pack completed:
   - Scope: Lobby and common areas only
   - Do-not-enter: Server room, executive offices
   - Time: Business hours only
   - Stop authority: Client security manager

3. Safety plan confirmed:
   - No forced entry
   - No impersonation
   - Emergency contact listed

4. Legal approval confirmed by client

5. Testing executed within scope

6. Report delivered:
   - Finding: Visitor badges not checked
   - Finding: Tailgating common
   - Recommendation: Badge verification training

7. Client implements improvements

## Key Terms Explained

| Term | Simple Definition |
|------|-------------------|
| **Physical Security Testing** | Authorized testing of facility access controls |
| **ROE** | Rules of Engagement—testing scope and rules |
| **Stop Authority** | Person who can halt testing immediately |
| **Do-Not-Do List** | Actions explicitly prohibited |
| **Tailgating** | Unauthorized person following authorized person through door |
| **Coercion** | Forcing someone to do something against their will |
