- Streamlines CMMC/NIST audit preparation
- Reduces audit stress and cost

## Step-by-Step: Compliance Support

### Step 1: Confirm Framework

Identify target requirements:
| Framework | Common For |
|-----------|------------|
| **HIPAA** | Healthcare |
| **PCI-DSS** | Payment processing |
| **SOC 2** | Service providers |
| **ISO 27001** | Security management |
| **Custom** | Client-specific needs |

### Step 2: Map Controls

For each requirement:

- Identify applicable controls
- Map to evidence sources
- Document gaps

### Step 3: Collect Evidence

Gather files for agreed period:
| Type | Examples |
|------|----------|
| **Access Records** | Who accessed what, when |
| **Change Records** | What changed, who approved |
| **Monitoring Logs** | Alerts and responses |
| **Backup Records** | Backup success logs |
| **Training Records** | Staff training completion |

### Step 4: Sanitize Files

- Remove sensitive data not needed for audit
- Scope-limit to relevant systems
- Ensure completeness

### Step 5: Assemble Evidence Pack

Create package with:
| Component | Description |
|-----------|-------------|
| **Table of Contents** | What's included |
| **Evidence Files** | Supporting documentation |
| **Gap Analysis** | What's missing |
| **Disclaimer** | Non-legal statement |

### Step 6: Include Disclaimer

Required statement:

> "Vantus provides operational support and evidence organization. We do not provide legal advice. Client retains full responsibility for compliance decisions and audit outcomes."

### Step 7: Create Fix Backlog

For identified gaps:

- Document remediation needed
- Prioritize by risk
- Track to closure

## Real Example

**Scenario**: Client needs evidence of access controls for audit.

**Compliance Support Process**:

1. Identify requirement: Access control evidence for PCI-DSS
2. Map to sources:
   - Access register
   - Change approval records
   - Quarterly access reviews

3. Collect files:
   - Access register (last 12 months)
   - Change records with approvals
   - Review meeting notes

4. Sanitize:
   - Remove unnecessary detail
   - Focus on relevant systems

5. Assemble pack:
   - Table of contents
   - Evidence files
   - Gap note: "Password policy review due, scheduled for next month"

6. Include disclaimer

7. Deliver to client

8. Create backlog item for password policy review
