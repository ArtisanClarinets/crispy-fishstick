## I. PURPOSE & SCOPE

This SOP defines the procedures for verifying that all systems are configured in accordance with the Vantus Systems Security Baseline. It applies to all production and pre-production systems.

## II. KEY PRINCIPLES

- **Automation:** Automate the verification of security baselines wherever possible.
- **Continuous Monitoring:** Continuously monitor systems for deviations from the security baseline.
- **Least Privilege:** Ensure that all systems are configured with the principle of least privilege.
- **Defense in Depth:** Use a layered approach to security to protect against a variety of threats.

## III. SECURITY BASELINE REQUIREMENTS

The Vantus Systems Security Baseline defines the minimum security requirements for all systems. It covers the following areas:

- **Access Control**
- **Cryptography**
- **Logging and Monitoring**
- **Network Security**
- **Operating System Hardening**
- **Web Security**

## IV. STEP-BY-STEP PROCEDURE

1.  **Define the Security Baseline:** The security team is responsible for defining and maintaining the security baseline.
2.  **Automate Baseline Checks:** Use automated tools to check for compliance with the security baseline.
3.  **Run Baseline Checks:** Run the baseline checks as part of the CI/CD pipeline and on a regular basis in production.
4.  **Report Deviations:** Report any deviations from the security baseline to the system owner.
5.  **Remediate Deviations:** The system owner is responsible for remediating any deviations from the security baseline.
6.  **Verify Remediation:** Verify that the deviations have been remediated.
7.  **Review and Update the Baseline:** Review and update the security baseline on a regular basis.

## V. TEMPLATES & CHECKLISTS

- **Security Baseline Checklist:** A checklist of all the requirements in the security baseline.
- **Baseline Deviation Report Template:** A template for reporting deviations from the security baseline.

## VI. ESCALATION PATHS

- If a deviation from the security baseline is not remediated in a timely manner, it will be escalated to the Engineering Manager.
- If the Engineering Manager does not take action, it will be escalated to the VP of Engineering and the CISO.
