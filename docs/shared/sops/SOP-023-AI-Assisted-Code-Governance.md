## I. PURPOSE & SCOPE

This SOP defines the governance and procedures for using AI-assisted coding tools at Vantus Systems. It aims to ensure that AI is used responsibly and effectively, without compromising code quality or security.

## II. KEY PRINCIPLES

- **Accountability:** Human developers are ultimately accountable for all code, whether it was written by them or by an AI.
- **Transparency:** All AI-generated code must be clearly identified.
- **Security:** Do not submit any sensitive or proprietary code to public AI services.
- **Quality:** All AI-generated code must be reviewed and tested to the same standards as human-written code.

## III. ALLOWED AI TOOLS

- **Gemini for Code:** Approved for general-purpose code generation and assistance.
- **GitHub Copilot:** Approved for use within VS Code.

## IV. STEP-BY-STEP PROCEDURE

1.  **Identify AI-Generated Code:** Add a comment to all AI-generated code, indicating which tool was used to generate it. For example: `// Generated by Gemini`.
2.  **Review AI-Generated Code:** All AI-generated code must be reviewed by a human developer. Do not trust AI-generated code to be correct or secure.
3.  **Test AI-Generated Code:** All AI-generated code must be accompanied by tests, just like human-written code.
4.  **Sanitize Inputs:** Before submitting any code to an AI tool, remove all sensitive information, such as API keys, passwords, and proprietary algorithms.
5.  **Audit AI Usage:** The use of AI coding tools will be audited on a regular basis to ensure compliance with this SOP.

## V. TEMPLATES & CHECKLISTS

- **AI Code Review Checklist:** A checklist of items to look for when reviewing AI-generated code.

## VI. ESCALATION PATHS

- If you are unsure whether it is safe to use an AI tool for a particular task, escalate to the Engineering Manager.
- If you discover a security vulnerability in an AI-generated code, report it immediately to the security team.
