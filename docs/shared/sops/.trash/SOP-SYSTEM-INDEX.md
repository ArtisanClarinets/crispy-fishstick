## VI. USING SOPs EFFECTIVELY

### For Executing a Process

1. **Find the SOP:** Use the domain summary (Section II) or quick navigation (Section V).
2. **Read Section II (Key Principles):** Understand the "why" before the "how."
3. **Follow Section IV (Step-by-Step Procedure):** Execute each phase in order.
4. **Use Templates & Checklists (Section VI):** Don't start from scratch.
5. **Check Escalation Paths (Section VII):** Know who to contact if something goes wrong.

### For Improving an SOP

1. **File an issue** in the `docs/sops/` folder.
2. **Describe the problem:** What's unclear? What doesn't work?
3. **Propose a change:** Be specific.
4. **Get feedback** from the SOP owner and affected stakeholders.
5. **Update the SOP:** Bump the version number (per SOP-000, Section IV).
6. **Announce to the team:** 2-week implementation window.

### For Understanding Exceptions

1. **Find the Exception Register** for your domain.
2. **Review recent waivers:** See what exceptions were granted and why.
3. **Request an exception** if you have a unique situation (see SOP-000, Section VII).
4. **CISO or domain owner approves** within 24 hours.

## IX. EXCEPTION HANDLING

**Exception Registers** track approved waivers to SOPs:

- Operations exception register — operations waiver log.
- [Exception Register: Engineering](exception-register-eng.md) — Engineering waiver log.
- [Exception Register: Security](exception-register-sec.md) — Security waiver log.
- [Exception Register: Business](exception-register-biz.md) — Commercial waiver log.
- [Exception Register: People](exception-register-peo.md) — People & Culture waiver log.

**How Exceptions Work:**

1. Identify that you need an exception to an SOP.
2. Document the reason: "Why can't we follow the normal process?"
3. Submit to the SOP owner or domain VP.
4. Owner responds within 24 hours: Approved, Approved with Conditions, or Denied.
5. If approved, log in the Exception Register (for audit trail).
6. Review exceptions quarterly to identify if the SOP should change.

## XIII. CONTINUOUS IMPROVEMENT

**How we improve the SOP system:**

### Monthly SOP Metrics Review

- SOP owner reviews audit logs.
- Identifies compliance gaps.
- Proposes improvements.

### Quarterly Domain Review

- VP of domain reviews all SOPs.
- Gathers feedback from team.
- Updates SOPs based on feedback.
- Announces changes to team.

### Annual Governance Review

- COO + all domain VPs review entire SOP system.
- Evaluates: Are SOPs effective? Are they being followed? Do new domains need SOPs?
- Updates SOP-000 framework if needed.
- Reports to CEO and board.
