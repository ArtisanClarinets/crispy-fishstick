## How to Use These Values

These are not wall decorations. They are decision-making tools.

When faced with a choice, run it through these values. If an option violates a value, it is automatically rejected. No exceptions.

**Example:** A client asks us to cut documentation to save money. Value 2 (Owner-Operability) says documentation is mandatory. We explain why. We do not compromise.

---

## Value 1: Tell the Truth

**What it means:** We are honest about complexity, risks, and our own limits.

**In practice:**
- If a task takes 40 hours, we say 40 hours. Not 20 to make the client happy.
- If a requested feature is a bad idea, we say so and explain why.
- If we make a mistake, we own it and fix it.

**Why it matters:** Trust is our currency. One lie destroys a hundred truths.

---

## Value 2: Build for Independence

**What it means:** We succeed when our clients no longer need us.

**In practice:**
- We write the disaster recovery runbook before we write the "nice-to-have" feature.
- We prioritize documentation over polish.
- We train client teams to operate without us.

**Why it matters:** Our goal is not job security through dependency. It is empowerment through capability.

---

## Value 3: Respect the Machine

**What it means:** We build software that respects hardware, networks, and human limits.

**In practice:**
- We do not ship 5-megabyte websites to users on slow connections.
- We test on real devices, not just fast office computers.
- We consider the person who will maintain this system at 2 AM.

**Why it matters:** Computers have limits. Networks have limits. Humans have limits. Good engineering respects all three.

---

## Value 4: Leave a Trail

**What it means:** Every significant action is documented and reversible.

**In practice:**
- We use Architecture Decision Records (ADRs) for major choices.
- We version-control infrastructure, not just code.
- We write runbooks for every critical process.

**Why it matters:** "How did we get here?" should never be a mystery.

---

## Value 5: Protect What Is Not Ours

**What it means:** We treat client budgets and security as if they were our own.

**In practice:**
- We recommend boring, stable technology over experimental frameworks.
- We question every expense: "Is this necessary?"
- We flag security risks even when fixing them costs us time.

**Why it matters:** Stewardship is a sacred trust. We are temporary custodians of things that matter deeply to our clients.

---

## Value 6: Speed Builds Trust

**What it means:** Performance is a feature. Slowness is a bug.

**In practice:**
- We hold ourselves to sub-2.5-second page loads.
- We target under 200 milliseconds for button responses.
- We optimize for the user on a three-year-old phone with spotty Wi-Fi.

**Why it matters:** Fast systems feel reliable. Reliable systems build confidence.

---

## Value 7: Design Is Engineering

**What it means:** Visuals are not decoration. They are the interface through which we communicate integrity.

**In practice:**
- We follow accessibility standards (WCAG 2.1 AA minimum).
- We use animation to guide attention, not distract from it.
- We design for clarity first, beauty second.

**Why it matters:** A confusing interface signals a confused system. Clear design signals clear thinking.

---

## Value 8: Humans Decide

**What it means:** Artificial intelligence helps. Humans remain responsible.

**In practice:**
- AI suggests code. Humans review and approve.
- AI finds patterns. Humans make decisions.
- If AI output conflicts with our standards, our standards win.

**Why it matters:** Tools amplify intent. They do not replace judgment.

---

## Value 9: Done Means Done

**What it means:** We do not ship half-finished work.

**In practice:**
- A feature is not done until it is tested, documented, and the client can use it.
- A project is not done until the handoff is complete.
- "Almost working" is not working.

**Why it matters:** Incomplete work creates debt. Debt slows everything down.

---

## Value 10: Small Business Deserves the Best

**What it means:** We bring the same rigor to a five-person shop that we bring to a Fortune 500.

**In practice:**
- No client is too small for our full attention.
- No budget is too small for our standard of care.
- We treat every business like the livelihood it is.

**Why it matters:** Small businesses are the backbone of the economy. They deserve technology that works as hard as they do.

---

## The Values in Action: Decision Matrix

| Scenario | Values Applied | Decision |
|----------|---------------|----------|
| Client asks to skip documentation to save money. | Value 2 (Independence) | Decline. Explain that docs are mandatory for their protection. |
| New framework promises faster development. | Value 5 (Stewardship) | Evaluate stability first. Prefer boring and working over new and risky. |
| Project deadline approaches. Tests are not passing. | Value 9 (Done Means Done) | Extend timeline or cut scope. Do not ship untested code. |
| AI suggests a code rewrite. | Value 8 (Humans Decide) | Human reviews before any merge. Verify against standards. |
| Client has a small budget. | Value 10 (Small Business) | Find a way to deliver value within constraints. Do not compromise quality. |
