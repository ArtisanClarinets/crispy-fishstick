## What This Document Is

This is our promise to you. It defines how we work, what you can expect, and the standards we hold ourselves to.

If we ever fall short of these commitments, we want to know. This covenant is meaningless if we do not live it.

---

## Commitment 1: Absolute Clarity

You will never wonder what is happening with your project.

### What This Means
- **Plain-language updates:** Every weekly report explains progress, risks, and decisions in terms you understand.
- **Honest trade-offs:** We present options with their true costs—time, money, and maintenance burden.
- **Auditability:** Every hour we bill connects to specific work you can see and verify.

### What You Get
- Weekly "State of the Build" reports every Friday.
- Access to project tracking tools (your choice: email, Slack, or project dashboard).
- Monthly video calls to review progress and adjust priorities.

### Our Standard
If you cannot explain what we did this week to a colleague, we have not done our job.

---

## Commitment 2: You Own Everything

We architect for your independence, not our retention.

### What This Means
- **Your accounts:** Cloud services, domains, and APIs are registered in your name with your email.
- **Your credentials:** You have root access from day one. We manage with your permission, not despite you.
- **Your exit:** Leaving Vantus is a documented process, not a negotiation.

### The Owner Handoff Package
At the end of every engagement, you receive:

| Item | Description |
|------|-------------|
| **Root Access Documentation** | How to access everything we built. |
| **Encrypted Key Backup** | All credentials in a secure vault you control. |
| **Runbooks** | Step-by-step instructions for every critical operation. |
| **Architecture Decision Records** | Why we built it this way. |
| **Exit Strategy** | How to transition to another vendor if needed. |

### Our Standard
If you cannot run your systems without us in 48 hours, we have not finished.

---

## Commitment 3: Security Is Standard

Security is not an add-on. It is the foundation.

### What This Means
- **Zero-trust baseline:** We assume attackers are already inside. Every request is verified.
- **Least privilege:** Access is granted on need, reviewed regularly, and revoked immediately when no longer needed.
- **No shortcuts:** We do not bypass security gates for speed or convenience.

### Security Impact Warnings
If you request something that compromises our security baseline:
1. We will explain the risk in plain language.
2. We will propose safer alternatives.
3. If you proceed anyway, we document the exception and mitigation steps.

### Our Standard
A system without security is not a Vantus system. Full stop.

---

## Commitment 4: Operational Readiness

We do not ship software. We ship systems that work in the real world.

### What This Means
- **Tested recovery:** Before launch, we perform a full disaster recovery drill—backup, restore, and verification.
- **Rollback plans:** Every high-risk change has a documented back-out plan tested before deployment.
- **Monitoring:** You can see system health in real-time. Alerts reach the right people before problems become outages.

### Production-Ready Checklist
Before any system goes live:
- [ ] Disaster recovery drill completed successfully.
- [ ] Rollback plan documented and tested.
- [ ] Monitoring dashboards configured.
- [ ] Alert thresholds set and validated.
- [ ] Runbooks written for common operations.
- [ ] On-call rotation established (if applicable).

### Our Standard
A system is not "production-ready" until we have proven it can recover from failure.

---

## Commitment 5: Documentation Is a Deliverable

Our documentation is as important as our code.

### What This Means
- **Living docs:** Documentation updates happen in the same pull request as the code it describes.
- **Version control:** Docs are stored with code, tracked in git, reviewed like code.
- **The 12-Month Test:** A junior engineer reading our docs 12 months from now should be able to maintain the system without calling us.

### Documentation Standards
Every project includes:
- **README:** Quick start and overview.
- **Architecture Guide:** How the system is organized.
- **API Documentation:** Every endpoint, parameter, and response.
- **Runbooks:** Operations procedures for common tasks.
- **Decision Records:** Why major choices were made.

### Our Standard
If it is not documented, it does not exist.

---

## Commitment 6: Fair Pricing

No surprises. No hidden fees. No games.

### What This Means
- **Fixed-price options:** Where scope is clear, we offer fixed-price contracts.
- **Time-and-materials transparency:** Where scope is exploratory, you see exactly what you are paying for.
- **No scope creep without consent:** Additional work requires explicit approval.

### Change Control
Any request outside the original scope triggers a Change Impact Assessment:
- Time impact (how much delay?).
- Cost impact (how much additional budget?).
- Risk impact (what new complexities arise?).

You approve before we proceed.

### Our Standard
You should never be surprised by an invoice.

## Commitment 7: Quality Guarantee

We stand behind our work.

### What This Means
- **Bug-free guarantee:** Critical bugs reported within 30 days of launch are fixed at no charge.
- **Performance targets:** If we miss agreed performance metrics, we fix it at no charge.
- **Security patches:** Critical security updates within 24 hours of notification.

### Warranty Period
- **30 days:** Bug fixes at no charge.
- **90 days:** Security patches at no charge.
- **12 months:** Documentation updates at no charge.

### Our Standard
If we built it wrong, we fix it. No argument. No delay.

1. **A designated decision-maker** who can approve trade-offs and answer questions.
2. **Timely access** to existing systems, documentation, and domain experts.
3. **Honest communication** about constraints, concerns, and changing priorities.
4. **Payment on time** per the agreed schedule.
