# Proof Page Spec

Version: 2.0
Last updated: 2026-03-06

## Purpose

Proof pages provide verifiable implementation outcomes without exposing internal-only data.

## Required sections

1. Problem context
2. Scope delivered
3. Evidence artifacts
4. Outcome summary
5. Next step CTA

## Evidence rules

- Evidence must be attributable and timestamped.
- Do not present unverifiable metrics.
- Internal security or infrastructure details are summarized only.
