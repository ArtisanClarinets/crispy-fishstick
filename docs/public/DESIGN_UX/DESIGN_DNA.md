# Design DNA (Public)

Version: 2.0
Last updated: 2026-03-06
Scope: visual and UX standards for `apps/public`

## Direction

- Tone: technical trust with plain-language clarity.
- Copy style: proof-driven and specific, never inflated claims.
- CTA style: clear next action (Start Audit or Contact).
- Visual mood: deep-navy precision, calm contrast, and ownership-first clarity.

## Foundation rules

- Typography is consistent and readable on mobile and desktop.
- Color tokens are centralized and reused across components.
- Motion is purposeful and reduced-motion safe.
- Contrast and focus treatment satisfy accessibility requirements.
- Brand rules are implemented from `public/DESIGN_UX/BRAND_GUIDELINES/`.

## Page pattern rules

- Hubs (`/services`, `/proof`, `/standards`, `/learn`) provide a clear path to detail pages.
- Detail routes include context, evidence, and one clear next action.
- Pricing route uses canonical offer names and ranges from pricing YAML.

## Public-safe scope rule

- This file defines the public presentation system only.
- Admin and portal UX/process details are not documented here.
