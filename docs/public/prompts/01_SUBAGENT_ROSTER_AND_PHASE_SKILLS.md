# Subagent Roster and Required Skills

Use this roster for dispatch. Load listed skills before each phase.

## Phase dispatch map

### Discovery and planning

- Subagent: architecture/research planner
- Skills:
  - `writing-plans`
  - `plan-protocol`
  - `next-best-practices`
- Output: canonical requirement matrix and conflict resolution table.

### System and route architecture

- Subagent: architecture + implementation planner
- Skills:
  - `nextjs-developer`
  - `next-best-practices`
  - `cache-components`
- Output: route architecture, server/client boundaries, caching and metadata plan.

### Visual system and UX

- Subagent: UX/design implementation
- Skills:
  - `frontend-design`
  - `web-design-guidelines`
  - `vercel-react-best-practices`
- Output: component patterns with desktop/tablet/mobile specifications and mobile-native interaction behaviors.

### Public route implementation

- Subagent: coding implementation
- Skills:
  - `nextjs-developer`
  - `vercel-react-best-practices`
  - `cache-components`
- Output: code in `apps/web` only, with route-by-route completion evidence.

### Security and hardening

- Subagent: security reviewer + implementer
- Skills:
  - `security-review`
  - `security-reviewer`
  - `secure-code-guardian`
- Output: security hardening report and fixed findings.

### QA and verification

- Subagent: QA engineer
- Skills:
  - `test-master`
  - `webapp-testing`
  - `verification-before-completion`
- Output: test evidence, responsive validation, accessibility and performance evidence.

### Release and handoff

- Subagent: docs/release coordinator
- Skills:
  - `requesting-code-review`
  - `create-pr`
  - `finishing-a-development-branch`
- Output: release packet, risk register, rollback triggers, handoff notes.

## Dispatch principles

- One phase at a time, each with explicit acceptance criteria.
- Parallelize only independent units.
- Every phase ends with command-level verification evidence.
