# Offer Ladder & Packages — Vantus Systems (v1.1)

> **Strategy:** Option A — Gulf Coast Leader
> Revenue Target: $500K Y1 → $1.4M Y2 → $2.2M Y3
> Target: Gulf Coast SMBs (5-50 employees)

**Date:** 2026-03-15

## The rule

If the offer isn’t productized, you can’t scale.

You will offer **three default packages** + modular add-ons. Anything else becomes a change request.

---

## Package 1: Website Rebuild (Modern + Complete)

**For:** SMBs with an outdated site that hurts trust.

**Includes (non-negotiable):**

- bespoke design implementation (based on provided brand/style input),
- responsive, fast, accessible site,
- **100% complete content** (built from content workbook),
- on-page SEO baseline,
- analytics instrumentation,
- hardened deployment,
- owner handoff package.

**Excludes (unless added):** custom portals, complex integrations.

**Acceptance criteria:**

- Lighthouse 95+ (mobile/desktop),
- WCAG 2.1 AA checks passed (with documented exceptions if any),
- content checklist 100% complete,
- security headers grade A (where applicable),
- handoff pack delivered.

---

## Package 2 (Core): Website + CMS

**For:** SMBs that need to update content weekly/monthly without calling you.

**Includes everything in Package 1, plus:**

- CMS installation (default: self-hostable headless CMS),
- content model for pages/posts/services,
- role-based access for editors,
- training session + short user guide.

**Acceptance criteria:**

- content editing works for non-technical user,
- staging → approval → publish workflow documented,
- rollback plan documented.

---

## Package 3: Website + Business Portal (Customer and/or Admin)

**For:** SMBs whose operations break on spreadsheets.

**Includes Package 2, plus:**

- secure auth,
- portal MVP (choose 1 primary workflow),
- audit log for key actions,
- integrations (one primary system),
- additional security hardening.

**Guardrail:** portals are scoped as **MVPs**, not enterprise re-platforming.

---

## Add-on modules (menu)

- Appointment/quote intake + routing
- CRM integration (HubSpot, Zoho, etc.)
- Payments (Stripe)
- File vault / document center
- Hardware purchase spec + on-site install
- Local deployment (on-prem) + monitoring
- Cloud deployment (VPS / managed cloud)

---

## How we prevent scope creep

- Every package includes a defined backlog.
- Anything outside backlog becomes a **Change Request** (see `SCOPING_AND_CHANGE_CONTROL.md`).
- “No content, no launch.” Content completion is a gate.

---

## Why this beats local competitors

- They ship brochures. You ship systems.
- They avoid backend work. You include CMS and optional portals.
- They promise. You provide proof artifacts.
