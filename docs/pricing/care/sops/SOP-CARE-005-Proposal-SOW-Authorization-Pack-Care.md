## What This Document Is

This SOP creates complete, clear agreements that match what we'll actually deliver. No surprises, no hidden clauses.

---

## Why This SOP Matters

This SOP defines a repeatable Vantus standard for the service area it covers.
Use it to create consistency, clear ownership, and auditable execution without unsupported market-comparison language.

---

## Why This Matters

A good agreement:

- Sets clear expectations
- Prevents scope disputes
- Documents what you own
- Defines response procedures
- Protects both parties
- **Enables CMMC audit readiness** through documented controls

---

## Three Core Principles

### 1. You Own Everything

- Your accounts
- Your data
- Your hardware
- We confirm this in writing

### 2. Recovery Beats Theory

- Contracts require schedules
- Rollback plans for changes
- Practical, not just theoretical

### 3. Consistent Structure

- Agreements reference catalog definitions
- No custom language that creates drift
- Same quality every time

## Step-by-Step: Creating Your Agreement

### Step 1: Start from Catalog

- Use standard module definitions
- No custom modules without Program Owner approval
- Pull from approved service catalog

### Step 2: Draft Statement of Work (SOW)

Your SOW includes:

| Section                     | What It Covers                  |
| --------------------------- | ------------------------------- |
| **Modules + Tier**          | Exactly what services you get   |
| **Schedule**                | When reports and reviews happen |
| **Evidence Files**          | What proof documents we deliver |
| **Do-Not-Do List**          | What's NOT included             |
| **Client Responsibilities** | What you need to provide        |
| **Escalation Paths**        | How to get help when needed     |

### Step 3: Include Scope Map

- Attach as appendix
- Or embed into agreement
- Defines boundaries clearly

### Step 4: Document Access Approvals

- Access authorization language
- Data handling expectations
- Evidence retention periods

### Step 5: Handle Security Testing

If authorized simulations are included:

- Require ROE (Rules of Engagement) Pack
- Define safety requirements
- Include stop authority provisions

### Step 6: Confirm Hardware Policy

- **NO hardware leasing language**
- Hardware is one-time purchase
- Separate invoice from services
- You own what you buy

### Step 7: Internal Review

| Role          | Review Focus              |
| ------------- | ------------------------- |
| Delivery Lead | Scope accuracy            |
| Program Owner | Policy compliance         |
| Security Lead | Security/simulation scope |

### Step 8: Store and Track

- Store signed agreement
- Create onboarding tickets
- Begin CHK-CARE-001 checklist

## When to Escalate

Escalate if:

- Legal/compliance language requested beyond operational support
- Client wants guarantees we can't provide
- Scope boundaries are unclear

**Always include disclaimers**: Vantus provides operational support, not legal advice.

## Audit & Compliance

**Monthly Check**:

- Every new client has:
  - Signed SOW
  - Scope Map
  - Access authorization on file
- Document any missing items
- Create corrective actions

---

## Real Example

**Scenario**: Client requests "PCI certification."

**Our Response**:

1. Offer **compliance support** (what we do)
2. Clarify client retains **certification responsibility** (what you do)
3. Include evidence packaging in scope
4. Document that we don't provide legal advice
5. Set realistic expectations
