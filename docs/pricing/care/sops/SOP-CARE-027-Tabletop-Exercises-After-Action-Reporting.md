## What This Document Is

This SOP delivers tabletop exercises and produces after-action reports with concrete improvements. Practice responses before real emergencies happen.

---

## Why Tabletop Exercises Matter

Tabletop practice:
- Tests decision-making without risk
- Identifies gaps safely
- Builds team coordination
- Creates improvement opportunities

---

## Two Core Principles

### 1. Proof Required
- After-action report documents what happened
- Specific changes identified
- Ownership assigned

### 2. Recovery Over Theory
- Focus on practical improvements
- Update real runbooks
- Build actual readiness

## Step-by-Step: Tabletop Exercise

### Step 1: Confirm Parameters
Define:
| Element | Description |
|---------|-------------|
| **Scenario** | What situation to practice |
| **Participants** | Who should attend |
| **Time Window** | How long the exercise |
| **Objectives** | What to learn |

### Step 2: Review Current Docs
Before exercise:
- Check current runbooks
- Note existing procedures
- Identify known gaps

### Step 3: Facilitate Exercise
During the session:
| Activity | Purpose |
|----------|---------|
| **Present Scenario** | Set the situation |
| **Pose Questions** | Drive decisions |
| **Track Decisions** | Document choices |
| **Identify Gaps** | Find missing pieces |

### Step 4: Capture Outcomes
Document:
- Decisions made
- Gaps identified
- Confusion points
- Improvement ideas

### Step 5: Produce After-Action Report
Report includes:
| Section | Content |
|---------|---------|
| **Scenario** | What we practiced |
| **Participants** | Who attended |
| **Key Findings** | What we learned |
| **Gaps** | What's missing |
| **Actions** | Specific improvements |

### Step 6: Implement Improvements
For each action:
- Assign owner
- Set due date
- Track completion

### Step 7: Verify Follow-Up
Schedule check:
- Did improvements happen?
- Are procedures updated?
- Is team better prepared?

## Real Example

**Scenario**: Ransomware tabletop exercise.

**Exercise Process**:
1. **Setup**:
   - Scenario: Friday 4 PM, ransomware detected
   - Participants: IT manager, CEO, office manager
   - Duration: 90 minutes

2. **Key Questions**:
   - Who decides to shut down systems?
   - How do we communicate with staff?
   - What's our recovery priority order?
   - When do we contact insurance?

3. **Findings**:
   - Gap: Unclear decision authority
   - Gap: No staff communication template
   - Gap: Recovery priorities not documented

4. **Actions**:
   - Define incident commander role
   - Create communication templates
   - Document system priority list

5. **Follow-up**: Verified all actions completed in 30 days
