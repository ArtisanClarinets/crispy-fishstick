## Why Monthly Reporting Matters

Regular scorecards:

- Show value delivered with proof
- Highlight issues early
- Track progress over time
- Enable data-driven decisions
- Demonstrate CMMC control maintenance

---

## Three Core Principles

### 1. Proof Required

- Every metric includes definition + timeframe
- No vague claims
- Data-backed statements

### 2. Plain Words

- Reports understandable to non-technical leaders
- Clear explanations
- No jargon

### 3. Standard Structure

- Same format every month
- Easy to compare
- Consistent metrics

## Step-by-Step: Monthly Scorecard

### Step 1: Collect Inputs

Gather monthly data:
| Category | Source |
|----------|--------|
| Changes | Change records |
| Incidents | Incident tickets |
| Monitoring | Alert summaries |
| Backups | Backup reports |
| Security | Security assessments |

### Step 2: Draft Scorecard

Using template TPL-CARE-003, include:

**Required Elements**:
| Section | Content |
|---------|---------|
| **Executive Summary** | What changed, what matters |
| **Incidents** | What happened, follow-ups |
| **Metrics** | Numbers with definitions |
| **Decisions Needed** | Client action items |
| **Next Actions** | What's coming, with dates |

### Step 3: Verify Definitions

Every metric includes:

- What it measures
- Time period covered
- How it's calculated

**Example**: "Uptime: 99.5% (measured as ping response success over 30 days)"

### Step 4: Review Internally

Check for:

- Scope accuracy
- Tone (no hype)
- Data correctness
- Action items clear

### Step 5: Deliver on Schedule

- Send to client on contracted date
- Store in client evidence folder
- Confirm receipt

### Step 6: Track Follow-Ups

Create tickets for:

- Decisions needed
- Next actions
- Owner assignments

## Real Example

**Good Scorecard Metric**:

> "Backup Success Rate: 98% (147 of 150 jobs successful, March 1-31, 2026)
>
> Failed jobs:
>
> - Server-03: 3 failures due to disk space
> - Fix planned: April 5, add storage
> - Owner: John Smith"

**Poor Scorecard Metric**:

> "Backups improved this month"
