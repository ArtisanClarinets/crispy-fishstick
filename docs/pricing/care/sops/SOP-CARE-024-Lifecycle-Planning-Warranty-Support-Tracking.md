## Step-by-Step: Lifecycle Management

### Step 1: Create Warranty Register
Track for critical assets:
| Information | Purpose |
|-------------|---------|
| **Asset Name** | What it is |
| **Purchase Date** | When acquired |
| **Warranty End** | When coverage expires |
| **Vendor** | Who provides support |
| **Support Level** | What's covered |

### Step 2: Define Refresh Horizons
By asset type, typical lifespans:
| Asset Type | Typical Lifespan |
|------------|------------------|
| **Laptops/Desktops** | 3-4 years |
| **Servers** | 4-5 years |
| **Network Equipment** | 5-7 years |
| **Storage** | 4-5 years |

### Step 3: Review Quarterly
Every quarter, check:
- Upcoming expirations (next 120 days)
- End-of-life announcements
- Assets approaching refresh horizon

### Step 4: Flag Expiries
When expiration within 120 days:
- Notify client
- Add to roadmap
- Present options

### Step 5: Recommend Replacements
Replacement plan includes:
| Element | Description |
|---------|-------------|
| **Options** | Good/better/best choices |
| **Tradeoffs** | Cost vs. features vs. timing |
| **Timeline** | When decision needed |
| **Budget** | Expected costs |

### Step 6: Update Roadmap
Add to quarterly review:
- Lifecycle decisions
- Budget planning
- Timeline updates

## Real Example

**Scenario**: Firewall warranty expires in 60 days.

**Lifecycle Process**:
1. Identify: Firewall-Warranty expires March 15
2. Assess: Risk of no vendor support
3. Research options:
   - Renew warranty: See ../pricing/CARE_PRICING_PUBLIC.md and current vendor renewal terms
   - Replace with new: $2,500
   - Accept risk: $0

4. Present to client in QBR:
   - Recommendation: Replace (firewall is 5 years old)
   - Timeline: Decision by February 1
   - Budget: $2,500 plus implementation

5. Client approves replacement

6. Add to roadmap: Firewall replacement project

7. Schedule: Order by Feb 15, install by March 1

| Term | Simple Definition |
