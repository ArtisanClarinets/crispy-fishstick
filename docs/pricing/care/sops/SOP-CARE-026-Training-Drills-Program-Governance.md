## What This Document Is

This SOP runs training focused on improvement—not punishment. We align drills to real risks your business faces.

---

## Why Training Matters

Effective training:
- Reduces human error
- Prepares staff for incidents
- Builds confidence
- Creates shared understanding

---

## Two Core Principles

### 1. Recovery Over Theory
- Practice builds readiness
- Reduces downtime during real events
- Practical beats perfect

### 2. Plain Words
- Training explains "why"
- No blame, no shame
- Learning-focused

## Step-by-Step: Training Program

### Step 1: Define Objectives
For each training:
| Element | Description |
|---------|-------------|
| **Learning Goals** | What participants should know/do |
| **Audience** | Who should attend |
| **Format** | Live, recorded, hands-on, etc. |

### Step 2: Plan Schedule
Publish:
| Element | Description |
|---------|-------------|
| **Quarterly or Annual** | Frequency based on contract |
| **Topics** | Security awareness, incident response, etc. |
| **Dates** | When sessions occur |

### Step 3: Use Plain Language
Materials should:
- Use everyday words
- Include real examples
- Focus on practical application
- Avoid technical jargon

### Step 4: Run Sessions
| Format | Best For |
|--------|----------|
| **Live Workshop** | Interactive skills |
| **Tabletop Exercise** | Decision-making practice |
| **Online Module** | Awareness topics |
| **Hands-On Drill** | Technical skills |

### Step 5: Document Outcomes
Capture:
- Attendance
- Key learnings
- Questions raised
- Improvement opportunities

### Step 6: Create Improvement Backlog
For gaps identified:
- Assign owners
- Set target dates
- Track to completion

### Step 7: Report Progress
Include in:
- Monthly scorecard
- Quarterly review
- Executive summaries

## Non-Punitive Approach

| We Do | We Don't |
|-------|----------|
| Focus on learning | Blame individuals |
| Reward improvement | Shame mistakes |
| Build skills | Create fear |
| Encourage questions | Intimidate |

## Real Example

**Scenario**: Run ransomware tabletop exercise.

**Training Process**:
1. **Objective**: Practice decision-making during ransomware incident

2. **Participants**: IT staff, executives, department heads

3. **Scenario**: Simulated ransomware attack on Friday afternoon

4. **Exercise**:
   - Walk through decisions
   - Practice communication
   - Test procedures

5. **After-Action**:
   - What worked well
   - What was unclear
   - Gaps identified

6. **Improvements**:
   - Update incident runbook
   - Clarify decision authority
   - Add communication templates

7. **Follow-up**: Schedule refresh in 6 months
