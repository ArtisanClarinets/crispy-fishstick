## What This Document Is

This SOP ensures nothing gets lost when you move from sales to delivery. Your requirements, constraints, and expectations transfer cleanly to the team that will serve you.

---

## Why This SOP Matters

This SOP defines a repeatable Vantus standard for the service area it covers.
Use it to create consistency, clear ownership, and auditable execution without unsupported market-comparison language.

---

## Why Handoffs Matter

A smooth handoff means:

- No repeated questions
- No lost information
- Clear expectations from day one
- Faster time to value (onboarding starts immediately)

---

## Two Core Principles

### 1. Plain Language

- Handoff memo is clear and unambiguous
- No assumptions or gaps
- Written for anyone to understand

### 2. Standard Setup

- Same onboarding tasks every time
- Same file structure
- Same quality checklist

| Role              | Responsibility                               |
| ----------------- | -------------------------------------------- |
| **Sales Lead**    | Provides fit notes and stakeholder map       |

| **Security Lead** | Confirms security scope and escalation rules |

## Step-by-Step: The Handoff Process

### Step 1: Create Handoff Memo

The memo includes:

| Section                | What It Covers                    |
| ---------------------- | --------------------------------- |
| **Goals**              | What you want to achieve          |
| **Constraints**        | Budget, timing, technical limits  |
| **Key People**         | Your contacts and decision makers |
| **Approvals Workflow** | Who signs off on what             |
| **Known Risks**        | Issues we're aware of upfront     |

### Step 2: Attach Signed Documents

- Signed agreement (SOW)
- Scope Map
- Any special provisions

### Step 3: Confirm Reporting Schedule

- Monthly scorecard? (if contracted)
- Quarterly review? (if contracted)
- Delivery timing expectations

### Step 4: Note Critical Dates

| Date Type      | Example                    |
| -------------- | -------------------------- |
| Renewals       | Software licenses, domains |
| Audits         | Compliance deadlines       |
| Seasonal Peaks | Busy business periods      |

### Step 5: Create Onboarding Tickets

- Standard ticket bundle created
- Tasks assigned
- Due dates set

### Step 6: Plan First Week

Access requests and password vault tasks scheduled immediately.

### Step 7: Confirm Hardware Policy

If hardware procurement involved:

- Confirm it's separate one-time purchase
- Not bundled into Care fees
- Document in handoff notes

## Real Example

**Scenario**: Sales promised "weekly updates."

**Handoff Action**:

1. Note the promise in handoff memo
2. Confirm actual contract terms:
   - Monthly scorecard included
   - Weekly updates NOT included
3. Delivery schedules monthly scorecard
4. Offers optional weekly check-in as add-on if desired
5. Documents what was promised vs. contracted
