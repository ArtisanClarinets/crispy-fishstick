## What to Expect

During your first 30 days, we:

- Set up secure access
- Document what you have
- Deploy monitoring (if included)
- Establish security baselines
- Review backup posture
- Deliver findings and roadmap
- **Create exit-ready documentation from day one**

---

## Four Core Principles

### 1. You Stay in Control

- Access set up under your accounts where possible
- You approve all major changes
- We work within your constraints

### 2. Do the Basics Right

- Security minimums established first
- Passwords properly stored
- Basic protections in place

### 3. Proof Required

- No "steady state" claims until baseline files exist
- Everything documented
- Before/after tracked

### 4. Recovery Over Speed

- Start safe, not fast
- Avoid risky "big bang" changes
- Document everything
- Tune over time

## Step-by-Step: Your First 30 Days

### Day 1-3: Kickoff

**What happens**:

- Confirm key contacts
- Define escalation tree
- Document approvals workflow
- Identify maintenance windows

**Your role**: Attend kickoff, provide contact list, approve schedule

### Day 4-7: Access & Vault Setup

**What happens**:

- Complete Access Setup (SOP-CARE-008)
- Set up password vault (SOP-CARE-009)
- Document who has access to what

**Your role**: Approve access requests, provide credentials securely

### Day 8-14: Inventory Baseline

**What happens**:

- Capture inventory (SOP-CARE-010)
- Document critical dependencies
- Record system ownership

**Your role**: Confirm unknown assets, verify ownership details

### Day 15-21: Monitoring (If Included)

**What happens**:

- Deploy monitoring agents
- Start alert catalog
- Begin tuning log
- Set up ticketing

**Your role**: Approve agent installation, confirm alert contacts

### Day 22-28: Backup Review (If Included)

**What happens**:

- Review backup posture
- Document retention expectations
- Create do-not-do list
- Note constraints

**Your role**: Approve backup scope, confirm retention needs

### Day 29-30: Patch Review (If Included)

**What happens**:

- Review patch/vulnerability status
- Set schedule and exceptions
- Document aging buckets

**Your role**: Approve patch windows, identify critical systems

### Day 30: 30-Day Findings

**What we deliver**:

- Risks identified
- Quick wins achieved
- Constraints documented
- Plain language summary

### Day 30: 90-Day Roadmap

**What we deliver**:

- Prioritized action list
- Decision points clearly marked
- Timeline with owners
- Expected outcomes

## When to Escalate

Escalate immediately if onboarding reveals:

- No backups in place
- Shared admin accounts
- Unknown system ownership
- Other unsafe conditions

**Action**: Deliver prioritized stabilization plan

## Real Example

**Scenario**: Client lacks Multi-Factor Authentication (MFA).

**Our Response**:

1. Document the risk in findings
2. Propose rollout plan with timeline
3. Do NOT proceed with high-risk access until agreed
4. Prioritize MFA implementation
5. Track in 90-day roadmap
