# CHK-CARE-013: Monthly Scorecard Checklist

**Document ID:** VS-CARE-CHK-013  
**Version:** 1.0.0  
**Effective Date:** 2026-02-21  
**Owner:** Account Manager  
**Status:** Active  
**Reading Level:** 8th Grade  
**Estimated Time:** 3-4 hours per scorecard  
**Risk Rating:** Medium (missed reporting erodes trust)

**Goal:** Deliver consistent, data-driven monthly reports that demonstrate value, highlight issues early, and enable informed business decisions.

---

## 1. Data Collection (Day 1-3 of month)

- [ ] Change records gathered (number of changes, success rate, rollback count)
- [ ] Incident data compiled (count by severity, MTTR, root causes)
- [ ] Monitoring metrics extracted (uptime %, alert count, noise ratio)
- [ ] Backup reports collected (success rate, RPO compliance)
- [ ] Security posture reviewed (patch status, vulnerability count, alerts)
- [ ] Project status updated (active projects, milestones, blockers)
- [ ] License/vendor renewals flagged (upcoming expirations)

**Time Estimate:** 60-90 minutes  
**Source Systems:** Ticketing, monitoring, backup, asset management  
**CMMC Alignment:** AU.2.1 (audit events), CM.2.2 (baseline compliance)

---

## 2. Metric Calculation & Verification

- [ ] Uptime percentage calculated (with definition: "measured as...")
- [ ] Incident count tallied (with severity breakdown)
- [ ] Mean Time to Repair (MTTR) computed (total downtime / incident count)
- [ ] Change success rate calculated (successful / total × 100)
- [ ] Backup success rate verified (completed jobs / scheduled jobs × 100)
- [ ] Patch compliance percentage confirmed (patched / total systems × 100)
- [ ] All metrics include time window ("for the period [date] to [date]")

**Time Estimate:** 30-45 minutes  
**Quality Check:** Every metric has definition + timeframe

---

## 3. Trend Analysis

- [ ] Current month vs. previous month (improvement or degradation?)
- [ ] Current month vs. same month last year (seasonal patterns?)
- [ ] Trend direction noted (getting better, worse, or stable?)
- [ ] Threshold breaches identified (SLA violations, target misses)
- [ ] Recurring issues flagged (same problem multiple months?)
- [ ] Improvement verification (last month's fixes worked?)

**Time Estimate:** 20-30 minutes  
**Vantus standard:** requiring year-over-year comparison

---

## 4. Narrative Drafting

- [ ] Executive Summary written (3-5 bullet points of what matters most)
- [ ] Key incidents summarized (what happened, impact, resolution)
- [ ] Changes explained (what we did, why, outcomes)
- [ ] Wins celebrated (improvements, milestones achieved)
- [ ] Concerns raised (issues needing attention, risks growing)
- [ ] Next month preview (planned work, scheduled maintenance)

**Time Estimate:** 30-45 minutes  
**Writing Standard:** 8th grade reading level, no jargon

---

## 5. Action Items & Decisions

- [ ] Client decision items identified (need client input/approval)
- [ ] Vantus action items listed (what we'll do, by when)
- [ ] Blockers highlighted (what's stopping progress)
- [ ] Risk items flagged (upcoming concerns)
- [ ] Owners assigned (every action has a name)
- [ ] Due dates set (every action has a date)

**Time Estimate:** 15-20 minutes

---

## 6. Internal Review

- [ ] Technical accuracy review (Delivery Lead verifies data)
- [ ] Tone check (no blame, professional, constructive)
- [ ] Scope accuracy (doesn't promise what's not contracted)
- [ ] Client-appropriateness (suitable for their technical level)
- [ ] Completeness check (all required sections present)
- [ ] Format consistency (follows TPL-CARE-003 structure)

**Time Estimate:** 20-30 minutes  
**CMMC Alignment:** CA.2.1 (security assessments)

---

## 7. Client Delivery

- [ ] Delivery date met (per contract SLA, typically by 5th business day)
- [ ] Distribution list correct (all stakeholders included)
- [ ] Format correct (PDF + editable if required)
- [ ] Evidence folder updated (scorecard archived)
- [ ] Receipt confirmed (client acknowledges delivery)
- [ ] Questions answered (follow-up within 24 hours)

**Time Estimate:** 15-20 minutes

---

## 8. Follow-Up Tracking

- [ ] Action items entered in ticketing system (tracked to completion)
- [ ] Decisions needed flagged for follow-up (client reminders)
- [ ] Blockers escalated if unresolved (management attention)
- [ ] Next month pre-work noted (early collection items)
- [ ] Feedback captured (client input on report quality)
- [ ] Process improvements logged (make next month easier)

**Time Estimate:** 15-20 minutes  
**Owner:** Account Manager

## Success Criteria

| Criterion | Target | Verification |
|-----------|--------|--------------|
| On-time delivery | 100% | Delivery timestamps |
| Zero data errors | 99%+ | Internal review |
| Client satisfaction | 90%+ | Survey feedback |
| Action item closure | 95%+ | Backlog review |
| Zero scope creep | 100% | Content audit |

---

## Common Pitfalls to Avoid

- **Vague metrics:** "Systems were stable" → "99.5% uptime (measured as...)"
- **Missing timeframes:** Always include "for the period..."
- **Jargon:** "MTR was suboptimal" → "Response time was slower than target"
- **Blame:** "Client didn't respond" → "Decision pending client input"
- **Surprises:** Flag issues immediately, don't wait for monthly report
