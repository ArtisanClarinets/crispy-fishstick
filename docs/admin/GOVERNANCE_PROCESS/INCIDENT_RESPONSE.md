# Admin Documentation - Incident Response

**Version:** 1.0.0
**Last updated:** 2026-03-08
**Applies to:** Admin systems and privileged operator workflows

## Severity levels

- SEV0: confirmed compromise, pricing-control abuse, or sensitive data exposure
- SEV1: major outage or high-risk vulnerability with likely exploitation path
- SEV2: partial outage or degraded admin capability
- SEV3: minor operational issue

## Roles

- Incident Commander
- Security Lead
- Ops Lead
- Comms Lead
- Scribe

## Lifecycle

1. detect
2. triage
3. contain
4. eradicate
5. recover
6. document follow-up actions

## Minimum admin playbooks

- privileged account compromise
- pricing or billing configuration error
- RBAC regression
- audit-log gap or logging failure
- internal service outage affecting operators
